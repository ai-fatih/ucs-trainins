# Быстрые ответы: ЕГАИС / Алкоголь

> Шаблоны ответов для тикетов поддержки. Основано на анализе ~2400 обращений.
> По каждому шаблону: **Когда возникает** → **Ответ** (готовый текст) → **Ссылки**

---

## 1. Порционный алкоголь — настройка

**Когда возникает:** Клиент жалуется, что не работает порционный учёт алкоголя; нет списания с марок; сканер не читает; не видит кнопку порционного алкоголя на кассе.

**Ответ:**
Для работы с порционным алкоголем необходимы настройки:
1. В **r_keeper**: включить порционную продажу в карточке блюда (вкладка «Алкоголь» → «Порционный»)
2. В **StoreHouse Pro**: проверить сопоставление товара ЕГАИС, убедиться, что марки сохранены в базе (запрос порционных остатков по справке Б)
3. Если сканер не читает — проверить настройки сканера (внешняя библиотека ExtdllEGAIS.dll в RK7)

**Ссылки:**
- https://docs.rkeeper.ru/rk7/latest/ru/portsionnaya-prodayoa-alkogolya-105877199.html
- https://docs.rkeeper.ru/sh5/egais-10815107.html

## 2. Порционный алкоголь — марка не списывается / ошибка при продаже

**Когда возникает:** При продаже АП кассир видит ошибку; марка не списывается; товар не пробивается.

**Ответ:**
1. Проверить запрос порционных остатков по справке Б — если марка белого цвета, она сохранена и доступна для работы
2. Если марка уже списана в 0 («серая»/«чёрная») — показать, как посмотреть остатки по запросу порц. остатков
3. Если к блюду не привязан комплект — рассказать, как должна быть оформлена карточка блюда (комплект обязателен для АП)
4. Проверить, не забыли ли поставить кегу на кран (для пива)

**Ссылки:**
- https://docs.rkeeper.ru/sh5/akt-spisaniya-10815523.html

## 3. Накладные ЕГАИС — не приходят / не подтверждаются

**Когда возникает:** Накладная от поставщика не отображается в StoreHouse; ошибка при подтверждении; нет связи с УТМ.

**Ответ:**
1. **Нет связи с УТМ** — проверить работу QushSvc.exe / QushMan.exe, восстановить соединение с УТМ
2. **Не проставлено сопоставление товаров** — выполнить сопоставление товаров ЕГАИС вручную
3. **Ошибка при активации** — в sdbman выставить лицензию на склад
4. Если работают параллельно с DinB — рекомендовать принимать накладную в DinB, либо обратиться в техподдержку для настройки

**Ссылки:**
- https://docs.rkeeper.ru/sh5/nyuansy-raboty-s-egais-v-storehouse-pro-10815109.html
- https://docs.rkeeper.ru/sh5/nastrojki-svyazi-s-utm-10815134.html

## 4. Акты списания — ошибка при отправке

**Когда возникает:** Акт списания не отправляется; ошибка 8004; марка не найдена в остатках.

**Ответ:**
1. **Ошибка 8004** — выполнить запрос порционных остатков по справке Б, проверить наличие списываемых марок на остатке
2. **Марка не читается** — убедиться, что марка отсканирована без лишних символов (возможно, русская раскладка)
3. **Автоматическое формирование** — если акт не формируется автоматически при продаже — проверить настройки авт. списания в карточке товара
4. **Отмена проведения** — если нужно отменить отправленный акт — выполнить отмену проведения документа

**Ссылки:**
- https://docs.rkeeper.ru/sh5/akt-spisaniya-10815523.html
- https://docs.rkeeper.ru/sh5/3ya-versiya-xml-v-egais-10815111.html

## 5. Пиво — розлив, кеги, кран

**Когда возникает:** Пиво не отображается на кассе; пропала кнопка пива; не получается поставить кегу на кран.

**Ответ:**
1. **Не отображается блюдо** — проверить, не привязан ли GTIN (глобальный идентификатор) на карточке блюда; если нет — привязать
2. **Пропала кнопка** — зайти в невидимые объекты, проверить, не снят ли товар с крана; поставить на кран
3. **Пиво б/а** — проверить заполнение карточки блюда для безалкогольного пива (не требует ЕГАИС, но может требовать ЧЗ)
4. **Списание пива** — консультация по списанию реализованного пива, проверка прихода в ЛК ЧЗ

**Ссылки:**
- https://docs.rkeeper.ru/sh5/egais-10815107.html

## 6. Возврат товара ЕГАИС

**Когда возникает:** Нужно оформить возврат алкогольной продукции поставщику.

**Ответ:**
1. Создать документ «Возврат товара поставщику» в StoreHouse
2. Указать причину возврата, выбрать накладную поступления
3. Отправить через УТМ в ЕГАИС
4. Дождаться подтверждения от поставщика
5. Если нет связи с УТМ — обратиться к администратору или в техподдержку

**Ссылки:**
- https://docs.rkeeper.ru/sh5/egais-10815107.html

## 7. Перемещение АП между подразделениями

**Когда возникает:** Нужно переместить алкогольную продукцию между точками; ошиблись КПП при приёме накладной.

**Ответ:**
1. Оформить «Расход-Приход» между подразделениями
2. Если накладная принята на неверный КПП — выполнить отмену для накладных и перенаправить
3. Убедиться, что у обоих подразделений настроена связь с УТМ

**Ссылки:**
- (см. документацию docs.rkeeper.ru по перемещениям)

## 8. Алкогольная декларация

**Когда возникает:** Отчёт «Алкогольная декларация» не формируется или больше не актуален.

**Ответ:**
Отчёт «Алкогольная декларация» больше не сдаётся в прежнем формате. Используйте отчёты в ЛК ФСРАР (ЕГАИС). Уточнить актуальные требования у бухгалтера клиента.

---

## 9. Ошибка «Некорректная дата/стамп даты документов»

**Когда возникает:** Ошибка в StoreHouse при работе с документами ЕГАИС — неверный формат даты.

**Ответ:**
1. Проверить системную дату на сервере с УТМ
2. В документах ЕГАИС дата должна быть не старше 3 дней
3. Проверить версию XML документов — рекомендуется v4
4. Выполнить синхронизацию времени

**Ссылки:**
- https://docs.rkeeper.ru/sh5/3ya-versiya-xml-v-egais-10815111.html
- https://docs.rkeeper.ru/sh5/4ya-versiya-xml-v-egais-19601004.html
