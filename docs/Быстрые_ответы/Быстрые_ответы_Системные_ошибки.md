# Быстрые ответы: Системные ошибки (РК, SH, СХ)

> Шаблоны ответов для тикетов поддержки. Основано на анализе ~1200 обращений.

---

## 1. Ошибка на кассе

**Когда возникает:** Кассир видит ошибку при пробитии позиции; чек не формируется; сбой в работе R_keeper.

**Ответ:**
1. Проверить карточку блюда в RK7 — настройки алкоголя, комплекта, GTIN
2. Если алкоголь ведётся через стороннюю систему (DinB и т.п.) — рекомендовать обратиться в их поддержку
3. Если ошибка связана с маркировкой — проверить остатки марок (запрос порц. остатков)
4. Если проблема с чеками — проверить подключение к ОФД

---

## 2. Не отображается блюдо / кнопка на кассе

**Когда возникает:** Позиция есть в меню, но не видна на кассе; блюдо пропало после обновления.

**Ответ:**
1. **Невидимые объекты** — зайти в настройки, отфильтровать по невидимым объектам, проверить не скрыта ли позиция
2. **Проверить наличие крана** (для пива/АП) — поставить на кран
3. **Проверить привязку GTIN** — если не привязан, позиция не отображается в списке
4. **Проверить категорию** — блюдо привязано к неверной группе/категории

---

## 3. Не работает сканер

**Когда возникает:** Сканер штрихкодов не читает марки; не распознаёт DataMatrix коды.

**Ответ:**
1. Убедиться, что сканер настроен на чтение DataMatrix (настройка сканера)
2. Проверить библиотеку ExtdllEGAIS.dll (для RK7) — наличие и версию
3. Если сканер USB — проверить драйверы COM-порта
4. Если сканер работает, но марка не находится — проверить, сохранена ли марка в базу СХ

---

## 4. Ошибка при активации накладной / документа

**Когда возникает:** Невозможно активировать накладную; документ не сохраняется с ошибкой.

**Ответ:**
1. Выставить лицензию на склад через sdbman
2. Проверить, не превышен ли лимит документов
3. Проверить формат даты (не старше 3 дней)
4. Если ошибка связана с ЕГАИС — проверить версию XML, наличие связи с УТМ

---

## 5. Завис / не отвечает StoreHouse

**Когда возникает:** SH5 не запускается; завис при выполнении операции; ошибка подключения к БД.

**Ответ:**
1. Перезагрузить сервер / компьютер
2. Проверить службу QushSvc.exe (серверный сервис)
3. Если не запускается — проверить логи в QushMan.exe
4. При ошибке БД — обратиться к администратору БД
5. Создать тикет в SD с приложенными логами

---

## 6. Не формируется Z-отчёт

**Когда возникает:** Z-отчёт не печатается; зажевало; повторно не распечатать.

**Ответ:**
1. Проверить подключение к принтеру чеков
2. Выполнить перезагрузку терминала
3. Если отчёт уже печатался — в RK7 можно распечатать дубликат Z-отчёта
4. Обновить драйвер ККТ

---

## 7. Сбой при обновлении / установке

**Когда возникает:** После обновления не работает RK7 / SH5; ошибки при запуске; несовместимость версий.

**Ответ:**
1. Откатить обновление до предыдущей версии
2. Проверить совместимость версий RK7, SH5 и УТМ
3. Выполнить обновление в sdbman вручную
4. Если ошибка сохраняется — создать тикет в SD, приложить скриншот ошибки и логи
