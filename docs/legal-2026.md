# Требования законодательства РФ к сайту — 2026

Сводка актуальных требований на 2026 год для сайтов со сбором данных, личным кабинетом, онлайн-записью, формами обратной связи и т.д.

---

## 1. 152-ФЗ «О персональных данных»

Базовый закон. Если на сайте есть хотя бы одна форма (заявка, регистрация, подписка, онлайн-запись), cookie/аналитика — сайт является оператором ПДн.

### Политика обработки ПДн / конфиденциальности (ст. 18.1)

- Отдельная страница, ссылка в футере на каждой странице сайта.
- Обязательные разделы: оператор (название, контакты), категории собираемых данных, цели обработки, правовые основания, сроки хранения, права субъекта, меры защиты, трансграничная передача (если есть), дата последнего обновления.

### Согласие на обработку ПДн (ст. 9)

**С 01.09.2025 — отдельный документ.** Больше нельзя прятать согласие внутри политики или пользовательского соглашения.

5 обязательных элементов:
1. ФИО (или иные данные для идентификации субъекта)
2. Наименование и адрес оператора
3. Конкретные цели обработки
4. Перечень ПДн, на которые даётся согласие
5. Перечень действий и способы обработки (сбор, запись, систематизация, накопление, хранение, уточнение, использование, передача, обезличивание, блокирование, удаление, уничтожение)

Дополнительно: срок действия согласия и порядок отзыва.

### Чекбокс согласия на формах

- Текст: «Я даю согласие на обработку моих персональных данных в соответствии с Политикой конфиденциальности» (со ссылкой на политику и на полный текст согласия).
- **Галочка не должна быть предустановлена** — только активное действие пользователя.
- Без согласия отправка формы невозможна (валидация на клиенте и сервере).
- **Техническая фиксация согласия**: сохранять дату/время, IP-адрес, user-agent — доказательство получения согласия.

### Уведомление Роскомнадзора (ст. 22)

- Подать до начала обработки ПДн через pd.rkn.gov.ru.
- Исключение: обработка только для исполнения договора с субъектом без передачи третьим лицам.
- При изменении целей/категорий данных — подать информационное письмо для актуализации.

### Локализация данных (ст. 18, ч. 5)

- Первичная база ПДн граждан РФ — на серверах, физически расположенных в России.
- Требование касается первичной записи, систематизации, накопления, хранения, уточнения, извлечения.
- Хостинг сайта и CRM — в РФ. Если используются зарубежные сервисы — первичная запись должна быть в РФ, далее трансграничная передача.

### HTTPS / SSL

- Шифрование при передаче данных — обязательно.
- TLS 1.3 (предпочтительно), TLS 1.2 (минимум).

### Назначение ответственного за ПДн (ст. 22.1)

Для организаций — приказом назначить лицо, ответственное за организацию обработки ПДн.

### Права субъекта (ст. 14–17)

- Право на доступ к своим данным — 10 рабочих дней
- Право на исправление
- Право на удаление / отзыв согласия — 30 дней
- Право на ограничение обработки
- Реализовать: раздел «Мои данные», кнопка «Удалить аккаунт», toggle отзыва согласия

### Уведомление об утечках (ст. 21, 23.1)

- 24 часа — уведомление РКН
- 72 часа — полный отчёт об инциденте
- Незамедлительно — уведомление субъектов (если нарушены их права)

## 2. Закон РФ «О защите прав потребителей» № 2300-1

### Реквизиты на сайте (ст. 9–10)

Для интернет-магазинов и сайтов, продающих услуги:
- Полное наименование юрлица / ФИО ИП
- ОГРН / ОГРНИП
- ИНН
- Юридический и фактический адрес
- Режим работы
- Лицензии (если деятельность лицензируется)

### Публичная оферта (ст. 437 ГК РФ)

Обязательна для интернет-магазинов и сервисов с оплатой: условия продажи, цены, порядок оформления, возврат.

### Сертификаты / декларации в карточках товаров — с 01.09.2026 (546-ФЗ)

- В описание товара добавить ссылку на запись о сертификате соответствия / декларации в реестре (реестр выданных сертификатов РФ или Единый реестр ЕАЭС).
- Если товар не подлежит обязательному подтверждению — указать это явно.
- Не распространяется на: товары б/у, продажу в торговом объекте с возможностью ознакомления.
- Штраф: до 50 000 руб.

## 3. 168-ФЗ «О государственном языке» — русский язык на сайте

**Вступил в силу с 01.03.2026.** Для всех B2C-сайтов (работа с физическими лицами).

### Что должно быть на русском

- **Интерфейс и навигация**: кнопки «Купить», «Добавить в корзину», «Оформить заказ», «Каталог», «Контакты», «Доставка», «О нас»
- **Фильтры и сортировка**: «Цена», «Популярность», «Сбросить»
- **Поля форм**: лейблы («Имя», «Телефон», «Email»), плейсхолдеры, сообщения об ошибках
- **Статусы заказов**: «В обработке», «Отправлен», «Доставлен»
- **Системные уведомления**: «Заказ оформлен», «Товар добавлен в корзину»
- **Описания товаров и услуг** — полностью на русском
- **Характеристики товаров**: размер, материал, состав, цвет
- **Условия доставки и оплаты**
- **FAQ**
- **Политика возврата и гарантийные условия**
- **Cookie-баннер**
- **SEO-метаданные**: Title, Description (желательно)
- **Alt-теги изображений**

### Что можно оставить на иностранном

- Фирменные наименования
- Товарные знаки, знаки обслуживания

### Как оформлять

- Русский текст — приоритетный, тем же шрифтом, размером и цветом, рядом с иностранным.
- Иностранные слова — только с равнозначным переводом.
- Термин «cookie» нужно пояснять: «файлы идентификации пользователей куки» (не закреплён в нормативных словарях).

## 4. Cookie и аналитика

- Cookie-баннер обязателен, если используются аналитические сервисы (Яндекс.Метрика, Google Analytics) или рекламные трекеры.
- Информирование + возможность отказаться от необязательных cookie.
- По 168-ФЗ — баннер на русском языке.
- Политика cookie — отдельно или в составе политики ПДн. Описать: типы cookie (технические, аналитические, рекламные), цели, сроки хранения, способы управления.

## 7. Чек-лист: что должно быть на сайте

### Обязательно для всех сайтов

- [ ] Контакты владельца (название, адрес, email) — в футере на каждой странице
- [ ] Политика конфиденциальности / обработки ПДн — отдельная страница, ссылка в футере
- [ ] Согласие на обработку ПДн — отдельный документ (с 01.09.2025)

### Если есть формы сбора данных (заявки, регистрация, подписка, онлайн-запись)

- [ ] Чекбокс согласия на каждой форме (не предустановлен)
- [ ] Ссылка на Политику рядом с чекбоксом
- [ ] Валидация: форма не отправляется без согласия
- [ ] Техническая фиксация согласия (дата, время, IP, user-agent)
- [ ] Отдельный чекбокс для рекламных рассылок (если есть)
- [ ] Собираются только необходимые данные (принцип минимизации)

### Если есть личный кабинет / регистрация

- [ ] Пользовательское соглашение — отдельная страница
- [ ] Реализовано: право на доступ, исправление, удаление данных
- [ ] Кнопка «Удалить аккаунт»
- [ ] Toggle отзыва согласия на обработку ПДн

### Если интернет-магазин / платные услуги

- [ ] Реквизиты (ОГРН/ОГРНИП, ИНН, адрес, режим работы)
- [ ] Публичная оферта
- [ ] Правила возврата и обмена
- [ ] Политика доставки
- [ ] Ссылки на сертификаты/декларации в карточках товаров (с 01.09.2026)

### Если используется аналитика / cookie

- [ ] Cookie-баннер на русском языке
- [ ] Возможность отказаться от необязательных cookie
- [ ] Политика cookie (отдельно или в составе политики ПДн)

### Русский язык (168-ФЗ, с 01.03.2026)

- [ ] Все кнопки, навигация, фильтры — на русском
- [ ] Лейблы и плейсхолдеры полей — на русском
- [ ] Сообщения об ошибках и уведомления — на русском
- [ ] Описания товаров/услуг — на русском
- [ ] FAQ, условия доставки/оплаты — на русском
- [ ] Cookie-баннер — на русском

### Организационные / compliance

- [ ] Уведомление РКН об обработке ПДн подано (pd.rkn.gov.ru)
- [ ] Ответственный за ПДн назначен (приказ)
- [ ] Внутренняя Политика обработки ПДн утверждена
- [ ] Модель угроз и акт классификации ИСПДн утверждены
- [ ] HTTPS включён (TLS 1.3 / 1.2)
- [ ] Хостинг / серверы — в РФ
- [ ] Аудит-лог действий с ПДн настроен
